# Plan: Fix Static Unfilled Circular Eyes on ESP32-S3 Display

## Objective
Replace `lv_line` wireframe eye rendering with filled polygon rendering to achieve animated, filled procedural robot eyes using LVGL.

## Current State
- Eyes render as static empty circles using `lv_line` objects
- `lv_line` only draws polylines (no fill capability)
- Inheritance from `SpiLcdDisplay` is correct
- Board parameters and LVGL init are properly configured
- Animation timer runs at 20 FPS but eyes don't update visually

## Root Cause
`lv_line` objects cannot be filled - they only draw outline polylines. Filled shapes require `lv_draw_polygon` or canvas-based rendering.

## Solution Overview
Switch from `lv_line_create` to `lv_canvas_create` with `lv_draw_polygon` for filled eye shapes while preserving existing animation logic.

#### Implement `UpdateEyePolygon()`
```cpp
void ProceduralDisplay::UpdateEyePolygon(lv_obj_t* canvas,
    const std::vector<cv::Point2f>& points,
    int16_t* point_buffer, size_t max_points) {
  
  DisplayLockGuard guard(this);
  
  // Clear canvas
  lv_canvas_fill_bg(canvas, lv_color_black(), LV_OPA_COVER);
  
  if (points.size() < 3) return;
  
  // Convert cv::Point2f to int16_t pairs
  size_t num_points = std::min(points.size(), max_points / 2);
  for (size_t i = 0; i < num_points; ++i) {
    point_buffer[i * 2] = static_cast<int16_t>(points[i].x);
    point_buffer[i * 2 + 1] = static_cast<int16_t>(points[i].y);
  }
  
  // Draw filled polygon with outline
  lv_draw_polygon_dsc_t poly_dsc;
  lv_draw_polygon_dsc_init(&poly_dsc);
  
  poly_dsc.points = point_buffer;
  poly_dsc.point_cnt = num_points;
  poly_dsc.bg_color = lv_color_white();      // Fill color
  poly_dsc.bg_opa = LV_OPA_COVER;            // Full opacity fill
  poly_dsc.outline_color = lv_color_black(); // Outline color
  poly_dsc.outline_width = 2;
  poly_dsc.outline_opa = LV_OPA_COVER;
  
  lv_canvas_draw_polygon(canvas, &poly_dsc);
  lv_canvas_send_to_front(canvas, false);
}
```

### Phase 3: Memory Considerations

#### Canvas Buffer Allocation
Two options for canvas buffers:

**Option A: Static buffers (preferred for embedded)**
```cpp
// In procedural_display.h
static constexpr size_t kCanvasBufferSize = DISPLAY_WIDTH * DISPLAY_HEIGHT * sizeof(lv_color_t);
uint8_t left_canvas_buffer_[kCanvasBufferSize];
uint8_t right_canvas_buffer_[kCanvasBufferSize];
```

**Option B: PSRAM allocation (if available)**
```cpp
// Allocate in constructor, free in destructor
left_canvas_buffer_ = static_cast<uint8_t*>(heap_caps_malloc(
    kCanvasBufferSize, MALLOC_CAP_SPIRAM));
```

Given memory constraints (31% free = ~2.2MB available), Option A with smaller canvas size is safer.

**Optimized canvas size:**
```cpp
// Use eye region size instead of full display
static constexpr int kEyeCanvasWidth = 120;
static constexpr int kEyeCanvasHeight = 80;
static constexpr size_t kEyeCanvasBufferSize = kEyeCanvasWidth * kEyeCanvasHeight * 2; // RGB565
```

### Phase 4: Alternative Approach (Lighter Weight)

If canvas memory is too large, use `lv_draw_polygon` directly on the screen:

```cpp
// In UpdateFaceGeometry():
DisplayLockGuard guard(this);

lv_layer_t* layer = lv_disp_get_layer_top(lv_disp_get_default());
lv_draw_polygon_dsc_t poly_dsc;
lv_draw_polygon_dsc_init(&poly_dsc);

// Configure polygon descriptor
poly_dsc.points = point_buffer;
poly_dsc.point_cnt = num_points;
poly_dsc.bg_color = lv_color_white();
poly_dsc.bg_opa = LV_OPA_COVER;
poly_dsc.outline_color = lv_color_black();
poly_dsc.outline_width = 2;

// Draw directly to layer
lv_draw_polygon(layer, &poly_dsc);
```

This avoids canvas buffers entirely but requires clearing previous frame.

## Risk Mitigation

### Risk 1: Memory overflow from canvas buffers
**Mitigation**: Use smaller canvas size (eye region only) or direct layer drawing

### Risk 2: Flickering from redraw
**Mitigation**: Use `lv_disp_t* disp = lv_disp_get_default(); lv_disp_flush_ready()` properly

### Risk 3: Performance degradation
**Mitigation**: Profile at 20 FPS; reduce point count if needed

### Risk 4: Breaking existing UI
**Mitigation**: Keep all non-eye UI elements unchanged; only modify eye rendering
