# Plan: Enable Procedural Robot Face Display

## Problem
Device shows old emoji-based face instead of new procedural robot eyes. Root cause: `CONFIG_USE_PROCEDURAL_DISPLAY` was not enabled in sdkconfig, and `ProceduralDisplay` was inheriting from `Display` instead of `SpiLcdDisplay`, so LVGL display driver was never registered.

## Analysis
- Kconfig option `USE_PROCEDURAL_DISPLAY` exists under `DISPLAY_STYLE` choice
- Default `DISPLAY_STYLE` is `USE_DEFAULT_MESSAGE_STYLE` 
- Board file conditionally compiles display based on `CONFIG_USE_PROCEDURAL_DISPLAY`
- `ProceduralDisplay` must inherit from `SpiLcdDisplay` (not `Display`) to get full LVGL + LCD driver initialization

## Changes Made

### 1. sdkconfig
- Set `CONFIG_USE_PROCEDURAL_DISPLAY=y`
- Set `# CONFIG_USE_DEFAULT_MESSAGE_STYLE is not set`

### 2. main/display/procedural/procedural_display.h
- Changed inheritance: `class ProceduralDisplay : public SpiLcdDisplay`
- Added LCD parameters to constructor: `offset_x, offset_y, mirror_x, mirror_y, swap_xy`
- Added LVGL object members: `left_eye_, right_eye_, left_points_, right_points_`
- Added UI members: `status_label_, chat_label_, bg_rect_`

### 3. main/display/procedural/procedural_display.cc
- Constructor now calls `SpiLcdDisplay` base with all LCD parameters
- Removed custom LVGL init (now handled by `SpiLcdDisplay`)
- `SetupUI()` creates LVGL objects on screen, sets up animation timer, plays boot clip
- All methods use `DisplayLockGuard` for thread safety
- `UpdateFaceGeometry()` generates polygon contours and updates LVGL line objects

### 4. main/boards/sp-esp32-s3-1.54-muma/sp-esp32-s3-1.54-muma.cc
- Updated `ProceduralDisplay` instantiation to pass all LCD parameters:
  ```cpp
  display_ = new procedural::ProceduralDisplay(panel_io, panel,
                                               DISPLAY_WIDTH, DISPLAY_HEIGHT,
                                               DISPLAY_OFFSET_X, DISPLAY_OFFSET_Y,
                                               DISPLAY_MIRROR_X, DISPLAY_MIRROR_Y,
                                               DISPLAY_SWAP_XY);
  ```
